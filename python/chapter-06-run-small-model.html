第6章

小さなモデルを動かしたい

ここでは、PyTorch のモデルに入力を通して、出力が返ってくる一連の流れを見ます。

今日は何をしたい？

model(x) という一行が、実際には「入力をモデルに通して答えを作る」動きだと分かることが目標です。まずはとても小さなモデルで、形がどう変わるかを追います。

最小コード

import torch
import torch.nn as nn

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

model = nn.Sequential(
    nn.Linear(4, 8),
    nn.ReLU(),
    nn.Linear(8, 2),
).to(device)

x = torch.randn(1, 4, device=device)
y = model(x)

print(x.shape)
print(y.shape)

入力が (1, 4) で、最後の出力が (1, 2) になっています。数字 4 個を受け取り、最後は 2 個の答えへ変えていると読めます。

どこがROCm？

モデルと入力の両方を GPU に送っておけば、下の重い計算は GPU 側で進みます。ROCm の見方では、この中に行列計算が何度も出ていると考えるとつながりやすいです。

ここで出てきたPython

  • オブジェクトを作って変数に入れる
  • nn.Sequential(...) のように部品を順につなぐ
  • model(x) のように、モデルを関数のように呼ぶ

よくあるつまずき

  • 入力の最後の大きさが Linear(4, 8) の 4 と合わないと止まります。
  • 入力だけ GPU、モデルは CPU のままだと device の不一致で止まることがあります。
  • ReLU は「マイナスを 0 にする簡単な部品」くらいに思えば十分です。
