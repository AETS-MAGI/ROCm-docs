第3章

表どうしを計算したい

行列の掛け算は、AI や GPU の話に何度も出てくる大事な形です。

今日は何をしたい？

NumPy の表どうしを掛けて、入力の shape と出力の shape がどう変わるかを読めるようにします。まずは「数字そのもの」より「表の形」に注目します。

最小コード

import numpy as np

a = np.array([[1, 2, 3],
              [4, 5, 6]], dtype=np.float32)
b = np.array([[7, 8],
              [9, 10],
              [11, 12]], dtype=np.float32)

c = a @ b

print(a.shape, b.shape, c.shape)
print(c)

a は (2, 3)、b は (3, 2) です。内側の 3 と 3 がつながるので、結果の c は (2, 2) になります。

どこがROCm？

この「表どうしを掛ける」計算は、あとで PyTorch の中でも何度も出てきます。ROCm の下では、こうした行列計算を速く回すためのライブラリが大事な役目を持っています。

ここで出てきたPython

  • @ で行列の掛け算を書く
  • 計算結果を新しい変数 c に受け取る
  • 複数の値をまとめて print() する

よくあるつまずき

  • (2, 3) と (2, 2) はそのままでは掛けられません。内側が合うかを先に見ます。
  • @ は「一個ずつ掛ける」ではなく、表どうしの掛け算です。
  • 結果の数よりも、まず出力 shape がどうなったかを見ると迷いにくくなります。

a や b の数字を入れ替えた上で、実行前に c.shape がいくつになるかを先に予想してみましょう。
