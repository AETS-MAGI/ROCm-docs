第9章

学習と推論の違いを見比べたい

似たコードに見えても、重みが動くかどうかで中身はかなり違います。

今日は何をしたい？

推論は「答えを出すだけ」、学習は「答えを出してから直すまで」です。この違いを、同じモデルを使った短いコードで見比べて、自分の言葉で説明できるようにします。

最小コード

import torch
import torch.nn as nn

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

model = nn.Linear(1, 1).to(device)
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)
x = torch.tensor([[1.0]], device=device)
y_true = torch.tensor([[2.0]], device=device)

before = model.weight.detach().clone()

model.eval()
with torch.no_grad():
    answer = model(x)

model.train()
loss = (model(x) - y_true).pow(2).mean()
optimizer.zero_grad()
loss.backward()
optimizer.step()

after = model.weight.detach().clone()

print("inference output:", answer.item())
print("weight changed:", not torch.equal(before, after))

このコードでは、前半が推論、後半が学習です。最後の weight changed が True なら、「学習側では重みが動いた」と読めます。

どこがROCm？

推論は forward だけ、学習は forward に加えて backward と update も入ります。ROCm の見方では、学習のほうが GPU にやらせる仕事が多い、と考えると整理しやすいです。

ここで出てきたPython

  • 同じ変数を前半と後半で使い分ける
  • before と after を比べる
  • not で真偽を反転して読む

よくあるつまずき

  • model.eval() だけでは「絶対に学習しない」わけではありません。学習しない流れは no_grad() とセットで見ると分かりやすいです。
  • 推論で答えを出しても、重みは変わりません。
  • 学習の後半だけを見ていると、何が増えたのかが見えにくいので、前半との対比が大切です。
