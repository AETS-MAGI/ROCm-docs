第1章

GPUが見えるか確かめたい

最初の一歩は、「この環境で GPU が使えるのか」を自分の目で確かめることです。

📌 この章を始める前に: Quickstart または 第0章 で ROCm と PyTorch のセットアップが済んでいることを前提にしています。まだの方は先にそちらを終わらせてから戻ってきてください。

今日は何をしたい？

ROCm 版 PyTorch が入っていて、いまの PC から GPU が見えているかを短いコードで確認します。まずここが通ると、この先の章で「CPU だけで動いていた」という遠回りを減らせます。

  • この章のゴールは 成功/失敗の判定 です（最適化はまだ不要）。
  • 迷ったら「HIP」「GPU available」「Device count」の3点だけ先に見ます。
  • 失敗しても普通です。次に打つコマンドまでこの章に書いてあります。

まずは実行前チェック（30秒）

  • いま使う Python 環境を決める（例: .venv/bin/python）。
  • pip show torch で torch が入っていることを確認する。
  • ROCm 環境なら、torch バージョンに +rocm が付くことが多いです。
    例: Version: 2.5.0+rocm6.2 → ROCm 版　／　Version: 2.5.0+cu121 → CUDA 版（AMD GPU では使えない）　／　Version: 2.5.0 → CPU 版の可能性

venv（仮想環境）について: .venv/bin/python はプロジェクトごとに分離した Python 環境を指します。単に python と打つと、システムに複数の Python が入っている場合にどれが動くか分かりにくくなります。パスを明示することで「今どの torch が入っているか」がはっきりします。

確認コード（そのまま実行版）

コードを見る前に一点: 下のコードには torch.cuda という名前が出てきますが、これは NVIDIA 専用ではありません。ROCm 版 PyTorch も同じ名前を使って GPU を操作します（詳しくは後の「どこがROCm？」で説明します）。また torch.version.hip は ROCm に含まれる HIP という部品のバージョンを取得しています。torch.cuda と torch.version.hip が混在しているのはそういう理由です。

import sys
import torch

print("Python:", sys.version.split()[0])
print("Torch:", torch.__version__)
print("HIP:", torch.version.hip)
print("GPU available:", torch.cuda.is_available())
print("Device count:", torch.cuda.device_count())

if torch.cuda.is_available():
    print("Device[0]:", torch.cuda.get_device_name(0))

この章では、コードそのものよりも 出てきた値をどう読むか が大事です。実行したら、まずは HIP と GPU available の 2 行を落ち着いて見ます。

実行コマンド（コピペ用）

# 例: venv の Python を使う場合
./.venv/bin/python -c 'import sys, torch; print("Python:", sys.version.split()[0]); print("Torch:", torch.__version__); print("HIP:", torch.version.hip); print("GPU available:", torch.cuda.is_available()); print("Device count:", torch.cuda.device_count()); print("Device[0]:", torch.cuda.get_device_name(0) if torch.cuda.is_available() else "(skip)")'

1行で実行できるので、「ファイルを作る前にまず確認したい」ときに便利です。
-c オプションについて: python -c '...' は、Python ファイルを作らずに引用符の中のコードを直接実行する書き方です。

実行すると何が出る？

Python: 3.14.3
Torch: 2.9.1+rocm6.3
HIP: 6.3.x
GPU available: True
Device count: 1
Device[0]: AMD Radeon RX Vega

このように出れば、まずは ROCm 版 PyTorch が入っていて、GPU も見えている と考えて大丈夫です。最後に GPU 名まで出ていれば、入口としてはかなり良い状態です。
注: Python や Torch のバージョン番号は環境によって異なります。数字が違っても問題ありません。Torch: xxxxxx+rocm... の形式で出ていれば ROCm 版と判断できます。

どう読めばいい？

  • HIP: に値がある → ROCm 版 PyTorch の可能性が高い
  • GPU available: True → PyTorch から GPU が見えている
  • Device count: 1以上 → 利用可能 GPU を列挙できている
  • HIP: None かつ False → ROCm 版ではない PyTorch の可能性がある
  • HIP: に値があるのに False → ROCm 版だが GPU が見えていない可能性がある

結果パターン別の次の一手

  • パターンA: HIPあり + True + count>=1 → この章はクリア。次章へ進んでOK。
    （GPU available: True であれば Device count は必ず 1 以上になります。）
  • パターンB: HIPあり + False → ROCm 版 PyTorch は入っているが、GPU がまだ見えていない状態です。
    これは「Python から見えない」問題で、まずシステム（ドライバ）レベルで GPU を認識しているか確認します。
    → ターミナルで rocminfo を実行し、出力の中に amdgpu や gfx で始まる行が出ていれば、ドライバは GPU を認識しています。それでも PyTorch から見えない場合は、ユーザを video / render グループに追加できているか確認します（groups コマンドで確認）。
  • パターンC: HIP None + False → torch 自体が ROCm 版ではない可能性があります。
    pip show torch を実行し、Version: の行を確認します。+rocm が含まれていれば ROCm 版、+cu なら CUDA 版（AMD GPU では動作しません）、何も付いていなければ CPU 版です。
    ROCm 版を入れ直す場合は PyTorch 公式サイトの ROCm 向けインストールコマンドを使います。
  • パターンD: True だが Device 名がおかしい → 一度 Python を再起動して再実行します。
    「おかしい Device 名」の例: llvmpipe、cpu、Virtio GPU など、AMD GPU の製品名らしくないものが出た場合です（正常例: AMD Radeon RX ...）。
    複数の Python 環境が混在している可能性があります。which python または which python3 で今どの Python を使っているか確認し、意図した venv の Python を明示的に指定して実行し直します。

どこがROCm？

ROCm 版 PyTorch でも、GPU の入口に torch.cuda という名前が残っています。ここはたしかにややこしいのですが、NVIDIA 専用という意味で読むのではなく、GPU を使う入口の名前がそのまま残っている と考えると分かりやすいです。ROCm 環境でも、ここを通って GPU の状態を見ます。

これは意図的な設計で、もともと CUDA 用に書かれたコードが AMD GPU でもなるべくそのまま動くように、ROCm 版 PyTorch は torch.cuda という名前の API をそのまま引き継いでいます。

HIP と ROCm の関係: ROCm はドライバ・ライブラリ・コンパイラなどをまとめた「大きなソフトウェアの集合体」です。HIP（ヒップ）はその中の一部品で、GPU 向けのプログラミング言語 API を担当します。torch.version.hip が返す値は HIP のバージョンです。「HIP のバージョンが分かる ≒ ROCm 環境にいる」と読むことができます。

ここで出てきたPython

  • import でライブラリを読み込む
  • print() で確認したい値を出す
  • if で「GPU が見えているときだけ」を分ける
  • torch.version.hip のように、ドットでつながる名前をたどって情報を読む
  • sys.version.split()[0] の読み方: sys.version は "3.11.9 (main, Apr 2024, ...)" のような長い文字列を返します。.split() で空白で分割してリストにし、[0] で先頭の要素（バージョン番号だけ）を取り出しています。

よくあるつまずき

  • GPU available: False と出た → いまは GPU が使えていない、という意味です。
  • HIP: None と出た → ROCm 版ではない PyTorch の可能性があります。
  • Device の行で止まった → GPU 名を取る前の段階で何か引っかかっています。
  • GPU 名が出なくても慌てず、まずは「HIP はどうか」「True / False はどうか」を順番に見ます。
  • /opt/amdgpu/.../amdgpu.ids: No such file or directory という警告だけ出る → 実行自体が成功していれば致命的ではありません（環境メッセージのことがあります）。

1分演習

次の順で確認してみましょう。GPU available、HIP、Device count の3つを紙にメモすると、次章以降で迷いにくくなります。

この3つをメモしておく理由: 次章からテンソル（数値の表）を GPU に送る操作が出てきます。そのとき「そもそも GPU が見えているのか」「ROCm 版なのか」を出発点として把握しておくと、エラーが出たときに「GPU の問題なのか・コードの問題なのか」を素早く切り分けられます。
