第1章

GPUが見えるか確かめたい

最初の一歩は、「この環境で GPU が使えるのか」を自分の目で確かめることです。

今日は何をしたい？

ROCm 版 PyTorch が入っていて、いまの PC から GPU が見えているかを短いコードで確認します。まずここが通ると、この先の章で「CPU だけで動いていた」という遠回りを減らせます。

  • この章のゴールは 成功/失敗の判定 です（最適化はまだ不要）。
  • 迷ったら「HIP」「GPU available」「Device count」の3点だけ先に見ます。
  • 失敗しても普通です。次に打つコマンドまでこの章に書いてあります。

まずは実行前チェック（30秒）

  • いま使う Python 環境を決める（例: .venv/bin/python）。
  • pip show torch で torch が入っていることを確認する。
  • ROCm 環境なら、torch バージョンに +rocm が付くことが多いです。

確認コード（そのまま実行版）

import sys
import torch

print("Python:", sys.version.split()[0])
print("Torch:", torch.__version__)
print("HIP:", torch.version.hip)
print("GPU available:", torch.cuda.is_available())
print("Device count:", torch.cuda.device_count())

if torch.cuda.is_available():
    print("Device[0]:", torch.cuda.get_device_name(0))

この章では、コードそのものよりも 出てきた値をどう読むか が大事です。実行したら、まずは HIP と GPU available の 2 行を落ち着いて見ます。

実行コマンド（コピペ用）

# 例: venv の Python を使う場合
./.venv/bin/python -c 'import sys, torch; print("Python:", sys.version.split()[0]); print("Torch:", torch.__version__); print("HIP:", torch.version.hip); print("GPU available:", torch.cuda.is_available()); print("Device count:", torch.cuda.device_count()); print("Device[0]:", torch.cuda.get_device_name(0) if torch.cuda.is_available() else "(skip)")'

1行で実行できるので、「ファイルを作る前にまず確認したい」ときに便利です。

実行すると何が出る？

Python: 3.14.3
Torch: 2.9.1+rocm6.3
HIP: 6.3.x
GPU available: True
Device count: 1
Device[0]: AMD Radeon RX Vega

このように出れば、まずは ROCm 版 PyTorch が入っていて、GPU も見えている と考えて大丈夫です。最後に GPU 名まで出ていれば、入口としてはかなり良い状態です。

どう読めばいい？

  • HIP: に値がある → ROCm 版 PyTorch の可能性が高い
  • GPU available: True → PyTorch から GPU が見えている
  • Device count: 1以上 → 利用可能 GPU を列挙できている
  • HIP: None かつ False → ROCm 版ではない PyTorch の可能性がある
  • HIP: に値があるのに False → ROCm 版だが GPU が見えていない可能性がある

結果パターン別の次の一手

  • パターンA: HIPあり + True + count>=1 → この章はクリア。次章へ進んでOK。
  • パターンB: HIPあり + False → ROCm 版は入っているが GPU が使えていない。まず rocminfo で GPU が見えるか確認。
  • パターンC: HIP None + False → torch 自体が ROCm 版ではない可能性。pip show torch でバージョン表記を確認。
  • パターンD: True だが Device 名が不自然 → 一度 Python を再起動して再実行。複数環境混在の可能性も確認。

どこがROCm？

ROCm 版 PyTorch でも、GPU の入口に torch.cuda という名前が残っています。ここはたしかにややこしいのですが、NVIDIA 専用という意味で読むのではなく、GPU を使う入口の名前がそのまま残っている と考えると分かりやすいです。ROCm 環境でも、ここを通って GPU の状態を見ます。

ここで出てきたPython

  • import でライブラリを読み込む
  • print() で確認したい値を出す
  • if で「GPU が見えているときだけ」を分ける
  • torch.version.hip のように、ドットでつながる名前をたどって情報を読む

よくあるつまずき

  • GPU available: False と出た → いまは GPU が使えていない、という意味です。
  • HIP: None と出た → ROCm 版ではない PyTorch の可能性があります。
  • Device の行で止まった → GPU 名を取る前の段階で何か引っかかっています。
  • GPU 名が出なくても慌てず、まずは「HIP はどうか」「True / False はどうか」を順番に見ます。
  • /opt/amdgpu/.../amdgpu.ids: No such file or directory という警告だけ出る → 実行自体が成功していれば致命的ではありません（環境メッセージのことがあります）。
