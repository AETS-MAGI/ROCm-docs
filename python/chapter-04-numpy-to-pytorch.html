第4章

NumPyからPyTorchへ移りたい

ここでは、「数字の表」という見方を PyTorch の tensor にそのままつなげます。

今日は何をしたい？

NumPy の配列と PyTorch の tensor が、どちらも shape や dtype を持つ「数字のまとまり」だと分かることが目標です。まずは似ているところを見て、あとで GPU に進みやすくします。

最小コード

import numpy as np
import torch

x_np = np.array([[1, 2],
                 [3, 4]], dtype=np.float32)
x_t = torch.tensor(x_np)

print(x_np.shape, x_np.dtype)
print(x_t.shape, x_t.dtype, x_t.device)

形はほぼ同じでも、PyTorch の tensor にはあとで GPU へ送れるという広がりがあります。いまはまず、shape と dtype を同じ気分で読めることが大事です。

どこがROCm？

NumPy 配列は基本的に CPU 側ですが、PyTorch tensor はそのまま GPU へ動かせます。ROCm の出番がはっきり見えてくるのは、この tensor が GPU に乗ったあとです。

ここで出てきたPython

  • 2 つのライブラリを import する
  • 変数名を分けて、NumPy と PyTorch の値を見比べる
  • .shape、.dtype、.device のような属性を読む

よくあるつまずき

  • PyTorch の tensor を作っただけでは、まだ GPU には乗っていません。
  • dtype の見え方が少し違っても、まずは「数字の種類」と分かれば十分です。
  • NumPy と PyTorch を別世界だと思わず、「表を持つ仲間」と考えると整理しやすくなります。

1分演習

NumPy を使わず、PyTorch だけで 2 行 3 列 の tensor を直接作って、shape と device を表示してみましょう。
