数字を表として持ちたい

ここでは NumPy を使って、数字の並びを「表」として見る感覚を作ります。

今日は何をしたい？

AI や GPU の話では、数字は 1 個ずつよりも「まとまった表」で出てくることが多いです。まずは NumPy 配列を作って、shape を「表の大きさ」として読めるようにします。

最小コード

import numpy as np

x = np.array([[1, 2, 3],
              [4, 5, 6]], dtype=np.float32)

print(x)
print(x.shape)
print(x.dtype)

この例では、2 行 3 列の表を 1 つのまとまりとして持っています。(2, 3) と見えたら、「2 行 3 列」と読めば大丈夫です。

どこがROCm？

この章のコードは CPU 側ですが、ここで作る「shape を読む習慣」がそのまま PyTorch と GPU の章につながります。ROCm でも、まずは何行何列なのかを読めることが大事です。

ここで出てきたPython

  • 入れ子の [] で表の形を作る
  • 変数 x にまとまりを入れておく
  • x.shape や x.dtype のように属性を読む

よくあるつまずき

  • 各行の長さがそろっていないと、きれいな表として扱いにくくなります。
  • (2, 3) を「2 列 3 行」と逆に読まないように気をつけます。
  • dtype は「数字の種類」です。最初は float32 と見えたら十分です。

数字を入れ替えて、3 行 2 列 の配列を 1 つ作ってみましょう。実行したら shape がどう表示されるかを見ます。
