Python and ROCm 教室

Python and ROCm Course

`__tmp/会話メモ.md` をもとにした章立ての叩き台

A curriculum draft based on the working memo in `__tmp/会話メモ.md`

このページは、Python 文法を先に全部教えるのではなく、ROCm / NumPy / PyTorch でやりたいことを順に達成しながら、必要な Python を回収するための叩き台です。最初は「GPU が見えるか確かめたい」から入り、最後は短い ROCm 実験コードを 1 本読めるところまでつなぎます。

This draft does not teach Python grammar in isolation first. Instead, it introduces Python through concrete ROCm / NumPy / PyTorch tasks, ending with the ability to read a short ROCm experiment script end to end.

設計ルール: 章名は `import` や `class` ではなく、`GPU が見えるか確かめたい`、`行列を計算したい`、`学習と推論の違いを見たい` のように「やりたいこと」で付けます。

Design rule: Chapter names are goal-first, not grammar-first. The learner starts from what they want to do, not from isolated Python topics.

この教室の見方: コードを読んで、その章で必要だった Python を後から整理します。文法は主役ではなく、ROCm の実験を進めるための道具として出てきます。

How to use this course: Read the code first, then collect the Python ideas that appeared. Python is treated as a tool for ROCm-oriented experiments, not as the main subject.

章の共通テンプレート

Shared Chapter Template

各章は、だいたい次の流れにすると読みやすくなります。

Each chapter can follow the same readable pattern.

  • 今日は何をしたい？ まず目的を 1 行で固定する。
  • What do we want to do today? Fix the goal in one line.
  • 最小コード まずは動く最小例を見る。
  • Minimum code Show the smallest working example first.
  • どこが ROCm？ ROCm / HIP / GPU 側で何が大事かを短く確認する。
  • Where is the ROCm part? Briefly point out the GPU-side or ROCm-side meaning.
  • ここで出てきた文法 その章で自然に出た Python を整理する。
  • Python that appeared here Collect the syntax and ideas that came up naturally.
  • よくあるつまずき `"cuda"` 表記や shape mismatch などを先回りして説明する。
  • Common stumbles Anticipate issues such as `"cuda"` names or shape mismatches.
  • 1 分演習 読んだ直後に 1 個だけ試す。
  • One-minute exercise End with one tiny task the learner can try immediately.

おすすめの入り方

Suggested Path

13 章全部を最初から一気に書く必要はなく、まずは最初の 5 章をそろえるだけでもかなり強い入口になります。

You do not need all 13 chapters at once. Even the first 5 chapters already create a strong entry path.

  1. GPU が見えるか確かめたい
  2. Check whether the GPU is visible
  3. 数字を表として持ちたい
  4. Hold numbers as tables
  5. 表どうしを計算したい
  6. Compute table-to-table math
  7. NumPy から PyTorch へ移りたい
  8. Move from NumPy to PyTorch
  9. データを GPU に送りたい
  10. Send data to the GPU

13 章の叩き台

13-Chapter Draft

`会話メモ.md` の流れを、公開ページ向けに読みやすく並べ直した版です。

A public-facing rewrite of the chapter flow proposed in the working memo.

1

GPU が見えるか確かめたい

Check Whether the GPU Is Visible

やること: `torch.version.hip`、`torch.cuda.is_available()`、`torch.cuda.get_device_name(0)` を読んで、ROCm 版 PyTorch と GPU の見え方を確認する。

Task: Confirm the ROCm build and GPU visibility using `torch.version.hip` and the usual device checks.

ここで出る Python: `import`、変数、`print`、`if`、文字列。

Python that appears: `import`, variables, `print`, `if`, and strings.

2

数字を表として持ちたい

Hold Numbers as Tables

やること: NumPy 配列を作って、`shape` を「表の大きさ」として読む。

Task: Create NumPy arrays and read `shape` as the size of a table.

ここで出る Python: list、`np.array`、基本代入、コメント。

Python that appears: lists, `np.array`, basic assignment, comments.

3

表どうしを計算したい

Compute Table-to-Table Math

やること: 行列積を試して、`@` と shape の読み方を結びつける。ROCm では rocBLAS / GEMM の入口として見る。

Task: Try matrix multiplication and connect `@` with shape reading. In ROCm terms, this is the entry point to GEMM-style thinking.

ここで出る Python: `@`、返り値、`dtype`、shape の整合確認。

Python that appears: `@`, return values, `dtype`, shape matching.

4

NumPy から PyTorch へ移りたい

Move from NumPy to PyTorch

やること: `torch.tensor` を見て、tensor を「GPU にも行ける数字の表」として理解する。

Task: Read `torch.tensor` as a number-table object that can also move to the GPU.

ここで出る Python: `.shape`、`.dtype`、`.device`、NumPy と tensor の比較。

Python that appears: `.shape`, `.dtype`, `.device`, and NumPy-vs-tensor comparison.

5

データを GPU に送りたい

Send Data to the GPU

やること: `torch.device("cuda" if ...)` と `.to(device)` を読んで、CPU と GPU の置き場所の違いを理解する。

Task: Use `torch.device(...)` and `.to(device)` to understand the difference between CPU and GPU placement.

ここで出る Python: `if`、メソッド呼び出し、エラーメッセージの読み方。

Python that appears: `if`, method calls, basic error reading.

6

小さなモデルを動かしたい

Run a Tiny Model

やること: `nn.Sequential`、`nn.Linear`、`nn.ReLU` を使った最小モデルで `model(x)` を読む。

Task: Read `model(x)` using a tiny `nn.Sequential` network.

ここで出る Python: オブジェクト、メソッド、関数っぽく呼べるモデル。

Python that appears: objects, methods, and callable model objects.

7

推論だけしたい

Do Inference Only

やること: `model.eval()` と `torch.no_grad()` を見て、「答えを出すだけ」のモードを理解する。

Task: Learn inference mode through `model.eval()` and `torch.no_grad()`.

ここで出る Python: `with`、`requires_grad`、状態切り替え。

Python that appears: `with`, `requires_grad`, and mode switching.

8

学習してみたい

Try Training Once

やること: `loss`、`optimizer.zero_grad()`、`backward()`、`step()` を 1 回だけ回して、重みが変わる感覚をつかむ。

Task: Run one minimal training step and see that weights actually change.

ここで出る Python: 連続したメソッド呼び出し、状態更新。

Python that appears: chained method calls and state updates.

9

学習と推論の違いを見比べたい

Compare Training and Inference

やること: `train()` / `eval()`、`no_grad()` の有無、重み更新の有無を対比して、「使っているだけでは賢くならない」を説明できるようにする。

Task: Compare training and inference clearly enough to explain why ordinary use does not update the model.

ここで出る Python: 比較表の読み方、条件ごとの見分け。

Python that appears: reading comparisons and distinguishing behaviors.

10

たたみこみを動かしてみたい

Try a Convolution

やること: `nn.Conv2d` と 4 次元 tensor を見て、画像っぽいデータと shape の読み方に慣れる。ROCm では MIOpen へつながる入口として見る。

Task: Use `nn.Conv2d` and a 4D tensor to get used to image-like shapes and the MIOpen-side mental model.

ここで出る Python: batch / channel の説明、shape 展開。

Python that appears: batch/channel interpretation and larger shape reading.

11

Attention をのぞきたい

Peek into Attention

やること: Q / K / V の shape、転置、`softmax`、`nn.MultiheadAttention` を見て、コードと式を結びつける。

Task: Connect Q / K / V shapes, transpose, `softmax`, and `nn.MultiheadAttention` with the matching formulas.

ここで出る Python: 多次元 shape、数式とコードの対応。

Python that appears: higher-dimensional shapes and code-to-math reading.

12

ログと shape を読めるようになりたい

Read Logs and Shapes

やること: `print`、f-string、shape mismatch、例外メッセージを使って、実験コードで止まったときに何がズレたか読めるようにする。

Task: Use `print`, f-strings, and shape errors to identify what went wrong in an experiment script.

ここで出る Python: f-string、簡単なデバッグ、例外メッセージの読み方。

Python that appears: f-strings, simple debugging, and exception reading.

13

小さな ROCm 実験コードを 1 本読めるようになりたい

Read One Small ROCm Experiment Script

やること: `import` 群、device 判定、tensor 作成、モデル、forward、場合によっては backward までを 1 本の短いコードでつなげて読む。

Task: Read one short script that includes imports, device selection, tensor creation, a model, forward, and possibly backward.

ここで出る Python: ここまでの内容をまとめて回収する。

Python that appears: a recap of everything introduced earlier.
