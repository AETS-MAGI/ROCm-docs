Python and ROCm 教室 13章ガイド

13-Chapter Guide to Python and ROCm

「何をしたいか」から進む学習ロードマップ

A learning roadmap organized by goals, not grammar topics

このページでは、Python と ROCm をつなぐ学習コースを 13 章で見渡せます。章名は「変数」「関数」ではなく、「GPU が見えるか確かめたい」「データを GPU に送りたい」「学習と推論の違いを見比べたい」のように、読む人の目的で並べています。

This page gives you the full 13-chapter learning path. Instead of grammar titles such as “variables” or “functions,” each chapter is named after a goal the learner actually wants to achieve.

このガイドの使い方: 最初から順番に読んでもよいですし、今の自分の疑問に近い章から入っても大丈夫です。あとで前の章に戻りながら読んでも、ちゃんとつながる流れになっています。

How to use this guide: You can read from the beginning or jump straight to the chapter closest to your current question. The flow is designed so that going back to earlier chapters still feels natural.

各章でそろえる型

What Each Chapter Should Include

各章は、次の 6 つがそろうと読みやすくなります。

Each chapter becomes much easier to follow when it includes these six parts.

  • 今日は何をしたい？ まず目的をはっきりさせる
  • What do we want to do today? Make the goal explicit
  • 最小コード いちばん小さく動く例を見る
  • Minimum code Show the smallest example that works
  • どこが ROCm？ GPU 側とのつながりを短く確認する
  • Where is the ROCm part? Briefly point out the GPU-side meaning
  • ここで出てきた Python 必要になった文法だけ整理する
  • What Python appeared here? Collect only the syntax that was actually needed
  • よくあるつまずき 読者が止まりやすい所を先回りする
  • Common mistakes Anticipate where beginners usually get stuck
  • 1 分演習 読んだ直後に小さく試す
  • Tiny exercise End with one small thing to try immediately

おすすめの流れ

Recommended Path

とくに最初の 5 章は、あとに続くページの土台になります。

The first five chapters are especially important because they support many later pages.

  1. GPU が見えるか確かめたい
  2. Check whether the GPU is visible
  3. 数字を表として持ちたい
  4. Hold numbers as tables
  5. 表どうしを計算したい
  6. Multiply tables
  7. NumPy から PyTorch へ移りたい

  9. データを GPU に送りたい
  10. Move data to the GPU

13 章の学習ロードマップ

The 13-Chapter Learning Roadmap

どの章も、コードと意味がつながることを大事にしながら進みます。

Each chapter is designed to connect code with meaning in a practical way.

1

GPUが見えるか確かめたい

Check Whether the GPU Is Visible

できるようになること: ROCm 版 PyTorch が動いているか、GPU が見えているかを自分で確かめられる。

You gain: the ability to check whether ROCm PyTorch is working and whether the GPU is visible.

ここで出る Python: `import`、変数、`print`、`if`。

Python that appears: `import`, variables, `print`, and `if`.

2

数字を表として持ちたい

Hold Numbers as Tables

できるようになること: NumPy 配列を「数字の表」として見て、shape を読めるようになる。

You gain: the habit of reading NumPy arrays as tables of numbers with shapes.

ここで出る Python: list、`np.array`、代入、コメント。

Python that appears: lists, `np.array`, assignment, comments.

3

表どうしを計算したい

Multiply Tables

できるようになること: 行列積を shape から読めるようになり、AI の基本計算とのつながりが見えてくる。

You gain: a shape-first understanding of matrix multiplication and its link to AI workloads.

ここで出る Python: `@`、返り値、`dtype`。

Python that appears: `@`, return values, `dtype`.

4

NumPyからPyTorchへ移りたい

Move from NumPy to PyTorch

できるようになること: tensor を「GPU にも行ける数字の表」として理解できる。

You gain: a practical mental model for tensors as number tables that can move to the GPU.

ここで出る Python: `.shape`、`.dtype`、`.device`。

Python that appears: `.shape`, `.dtype`, `.device`.

5

データをGPUに送りたい

Move Data to the GPU

できるようになること: CPU と GPU の置き場所の違いを理解し、`.to(device)` を読めるようになる。

You gain: a clear understanding of CPU vs GPU placement and how `.to(device)` is used.

ここで出る Python: `if`、メソッド呼び出し、簡単なエラーの読み方。

Python that appears: `if`, method calls, and simple error reading.

6

小さなモデルを動かしたい

Run a Small Model

できるようになること: `nn.Sequential` や `model(x)` の読み方がわかる。

You gain: the ability to read `nn.Sequential` and understand what `model(x)` means.

ここで出る Python: オブジェクト、メソッド、関数のように呼べるモデル。

Python that appears: objects, methods, and callable models.

7

推論だけしたい

Do Inference Only

できるようになること: `eval()` と `no_grad()` を見て、答えを出すだけのモードを理解できる。

You gain: a clear picture of what inference mode means through `eval()` and `no_grad()`.

ここで出る Python: `with`、状態切り替え。

Python that appears: `with` and mode switching.

8

学習してみたい

Train Once

できるようになること: `loss`、`backward()`、`step()` が重み更新につながることを見られる。

You gain: a first view of how `loss`, `backward()`, and `step()` actually change model weights.

ここで出る Python: 連続したメソッド呼び出し、更新の流れ。

Python that appears: chained method calls and update flow.

9

学習と推論の違いを見比べたい

Compare Training and Inference

できるようになること: 「使うだけでは学習しない」を、自分の言葉で説明しやすくなる。

You gain: the ability to explain why ordinary model use is not the same as training.

ここで出る Python: 比較の読み方、条件ごとの差。

Python that appears: comparison reading and difference spotting.

10

たたみこみを動かしてみたい

Run a Convolution

できるようになること: `Conv2d` と画像っぽい 4 次元 tensor を shape で追えるようになる。

You gain: the ability to follow `Conv2d` and image-like 4D tensors through their shapes.

ここで出る Python: batch、channel、shape 展開。

Python that appears: batch, channel, and larger shape reading.

11

Attentionをのぞきたい

Peek into Attention

できるようになること: Q / K / V、転置、softmax をコードと shape で見比べられる。

You gain: the ability to compare Q / K / V, transpose, and softmax through code and shapes.

ここで出る Python: 多次元 shape、コードと式の対応。

Python that appears: higher-dimensional shapes and code-to-math reading.

12

ログとshapeを読めるようになりたい

Read Logs and Shapes

できるようになること: shape mismatch や簡単なログから、「どこがずれたか」を見つけやすくなる。

You gain: the ability to spot what went wrong from shape mismatches and simple logs.

ここで出る Python: `print`、f-string、簡単なデバッグ。

Python that appears: `print`, f-strings, and basic debugging.

13

小さなROCm実験コードを1本読めるようになりたい

Read One Small ROCm Experiment Script

できるようになること: device 判定、tensor 作成、モデル、forward、場合によっては backward までを 1 本の短いコードとして追える。

You gain: the confidence to follow a short script that includes device selection, tensors, a model, forward, and possibly backward.

ここで出る Python: それまでの内容をまとめて使う。

Python that appears: a recap of everything introduced earlier.

大事な考え方: Python の文法は、このコースでは主役ではありません。やりたいことを 1 つずつ達成する中で、必要な分だけ自然に覚えていく方針です。

Core idea: Python grammar is not the main character here. The course teaches it naturally as needed while solving concrete ROCm-related tasks.

はじめて読む人へ: 難しそうに見える章があっても、全部を一気に理解する必要はありません。今の自分の疑問に近い章から入って、あとで戻る読み方で大丈夫です。

For first-time readers: even if some chapters look advanced, you do not need to master everything at once. It is fine to start from the chapter that matches your current question and come back later.

気になる章から入って、必要になったら前の章へ戻る読み方でも大丈夫です。 It is perfectly fine to start from the chapter you need and return to earlier ones when necessary.
