第7章

推論だけしたい

ここでは、学習はせずに「答えを出すだけ」の流れを切り分けて見ます。

今日は何をしたい？

推論は、できあがったモデルに入力を通して答えを受け取る場面です。ここでは eval() と no_grad() を使って、「学習はしないで使うだけ」を短い形で確認します。

最小コード

import torch
import torch.nn as nn

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

model = nn.Sequential(
    nn.Linear(4, 8),
    nn.ReLU(),
    nn.Linear(8, 2),
).to(device)

x = torch.randn(1, 4, device=device)

model.eval()
with torch.no_grad():
    y = model(x)

print(y)

ここでやっているのは、入力から答えを出す forward だけです。重みを直したり、学習したりはしていません。

どこがROCm？

モデルと入力が GPU 上にあれば、推論の計算は GPU 側で動きます。ROCm の見方では、推論は「答えを出すだけ」なので、学習より仕事が少ない流れです。

ここで出てきたPython

  • with ...: の形で、一時的なモードを作る
  • メソッドを順番に呼んで動作を切り替える
  • 最後の結果だけを受け取って表示する

よくあるつまずき

  • model.eval() は「推論向けの設定へ切り替える」ための一歩です。
  • torch.no_grad() を忘れても動くことはありますが、推論だけの場面では入れておくと意図が伝わります。
  • 「答えを出した」ことと「学習した」ことは別です。この章ではまだ学習していません。
